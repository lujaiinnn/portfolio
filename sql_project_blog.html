Sales SQL Database Project

"Tracking Sales and Analyzing Business Performance with SQL"

The Sales SQL Database Project is designed to simulate a sales environment that captures customer, product, region, and transactional data. The database allows businesses to track sales activity, analyze product performance, and make data-driven decisions.

The project aims to design a fully functional relational database using MySQL to store and manage sales-related data. By utilizing SQL queries, we can gain insights into key metrics such as total revenue, product performance, customer behavior, and regional sales trends.

The key objectives of this project include:

  • Track customer purchases over time
  • Determine top-performing products
  • Analyze sales performance by product and region
  • Identify peak revenue periods
  • Enable decision-making based on customer behavior and trends

The database structure consists of four main tables:

  • Customers: Tracks customer details
  • Regions: Represents geographical market regions
  • Products: Stores information about products like name, category, and price
  • Sales: Captures transaction data, linking customers and products

Here is an overview of the table structure:

Regions Table

      CREATE TABLE Regions (
        region_id INT PRIMARY KEY,
        region_name VARCHAR(100)
      );

Customers Table

      CREATE TABLE Customers (
        customer_id INT PRIMARY KEY,
        name VARCHAR(100),
        region_id INT,
        FOREIGN KEY (region_id) REFERENCES Regions(region_id)
      );

Products Table

      CREATE TABLE Products (
        product_id INT PRIMARY KEY,
        product_name VARCHAR(100),
        category VARCHAR(100),
        price DECIMAL(10,2)
      );

Sales Table

      CREATE TABLE Sales (
        sale_id INT PRIMARY KEY,
        product_id INT,
        customer_id INT,
        quantity INT,
        sale_date DATE,
        FOREIGN KEY (product_id) REFERENCES Products(product_id),
        FOREIGN KEY (customer_id) REFERENCES Customers(customer_id)
      );

The database supports various SQL queries for generating insights such as:

  • Total revenue generated
  • Revenue by month
  • Top-selling products
  • Revenue by region
  • Customer purchase behavior analysis

For example, to calculate the total revenue:

      SELECT SUM(p.price * s.quantity) AS total_revenue
      FROM Sales s
      JOIN Products p ON s.product_id = p.product_id;

Additional insights from the data include:

  • Peak Sales Months
  • High-Performing Products
  • Customer Segmentation
  • Regional Demand Analysis

The project can be further extended by adding features like inventory management, supplier data, or integration with business intelligence tools like Power BI or Tableau.

Conclusion: The Sales SQL Database Project showcases the power of relational databases and SQL in managing and analyzing business data. It provides valuable insights into customer behavior, sales performance, and market trends, making it an essential tool for data-driven decision-making.
